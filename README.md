# chromesdaddy

Сервис запускается в докер контейнере и предоставляет клиентам интерфейс для взаимодействия с инстансами хром браузера через Chrome DevTools Protocol,
управляет множественными клиентскими соединениями и запуском/остановкой хрома для каждого клиента.

1. В начале работы клиент посылает на сервис инит запрос с опциональными параметрами запуска хрома;
2. Сервис запускает хром и возвращает его ID клиенту;
3. Клиент делает запрос для установления соединения с выбранным браузером. 

##### Запуск в докере

`docker build . -t chromesdaddy`

`docker run --rm -it -p 9222:9222 -e PORT=9222 chromesdaddy`

Для демонстрации работы после запуска контейнера можно запустить ./_testClient/main.go
